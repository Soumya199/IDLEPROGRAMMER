Abstract class and Interface in C#

Abstract Class in C# :-
  • Abstract Class is a restricted class and it does not allow you to create its object.
  • An abstract class can contain both abstract and nonabstract methods in it.
  • As Abstract class does not allow you to create an object of it so we can inherit it by the subclass that either implements or override its method.
  • It cannot be sealed
  • It may contain abstract and non-abstract function members
  • It is a part of OOPS

Syntax: 

        
            abstract class Test
            {{'{'}}
                //Your codes here
            {{'}'}}
            // class 'Test' is abstract

Example: 

        
            using System.IO;
            using System;

            abstract class Example
            {{'{'}}
                public abstract void First();
                public void Second()
                {{'{'}}
                    Console.WriteLine("I am a non abstract method");
                {{'}'}}
            {{'}'}}
            class Inheritor:Example
            {{'{'}}
                public override void First()
                {{'{'}}
                    Console.WriteLine("I am an abstract method");
                {{'}'}}
            {{'}'}}
            class Program
            {{'{'}}
                static void Main()
                {{'{'}}
                    Inheritor obj=new Inheritor();
                    obj.First();
                    obj.Second();
                {{'}'}}
            {{'}'}}

            Output :-
            I am an abstract method
            I am a non abstract method

Interface in C# :-

  • An interface defines a contract.
  • Interface is a purely abstract class; it has only signatures, no implementation.
  • May contain methods, properties, indexers and events.
  • Interface members are implicitly public abstract (virtual).
  • Interface members must not be static.
  • Classes and structs may implement multiple interfaces.
  • Interfaces can extend other interfaces.

Syntax: 

        
            interface IBaseClass1
            {{'{'}}
                void MethodA();
            {{'}'}}
            interface IBaseClass2
            {{'{'}}
                void MethodB();
            {{'}'}}
            class IMyInterface: IBaseClass1, IBaseClass2
            {{'{'}}
                void MethodA()
                {{'{'}}
                    //Your Method Defination goes here
                {{'}'}}
                void MethodB()
                {{'{'}}
                    //Your Method Defination goes here
                {{'}'}}
            {{'}'}}

Example: 

        
            using System.IO;
            using System;

            interface IFirst
            {{'{'}}
                void First();
            {{'{'}}
            interface ISecond
            {{'{'}}
                void Second();
            {{'}'}}
            class Implementor:IFirst,ISecond
            {{'{'}}
                public void First()
                {{'{'}}
                    Console.WriteLine("This is from First Interface");
                {{'}'}}
                public void Second()
                {{'{'}}
                    Console.WriteLine("This is from second Interface");
                {{'}'}}
                
            {{'}'}}
            class Program
            {{'{'}}
                static void Main()
                {{'{'}}
                    Implementor obj=new Implementor();
                    obj.First();
                    obj.Second();
                {{'}'}}
            {{'}'}}

            Output :
            This is from First Interface
            This is from second Interface
Difference between Abstract class And Interface :-

Abstract Class Interface
It contains both the declaration and definition parts. It contains only a declaration part.
Multiple inheritance is not achieved by an abstract class. Multiple inheritance is achieved by the interface.
It contains a constructor. It does not contain a constructor.
It can contain static members. It does not contain static members.
A class can only use one abstract class. A class can use multiple interface.
An abstract class can contain constants, methods, fields, etc. The interface can only contain methods.
It can be fully, partially, or not implemented. It should be fully implemented.
t can contain access modifiers like public, private, protected, etc. It contains only public access modifier because everything in the interface is public.

How to Implement Multiple Interfaces Having Same Method Name in C#

This is a question that often asked in interviews.
Let us consider an example and learn how to do that.

Example: 

        
            using System.IO;
            using System;

            public interface ISame1
            {{'{'}}
                void Print();
            {{'}'}}
            public interface ISame2
            {{'{'}}
                void Print();
            {{'}'}}

            class Program : ISame1, ISame2
            {{'{'}}
                void ISame1.Print()
                {{'{'}}

                    Console.WriteLine("Print method from ISame1");
                {{'}'}}
                void ISame2.Print()
                {{'{'}}

                    Console.WriteLine("Print method from ISame2");
                {{'}'}}
                static void Main()
                {{'{'}}
                    Program p = new Program();
                    ((ISame1)p).Print();
                    ((ISame2)p).Print();
                {{'}'}}
            {{'}'}}

            Output :
            Print method from ISame1
            Print method from ISame2
  • First, we are creating two interfaces with name ISame1 & ISame2, both of two contains the same method named as Print method.
  • Now we have to implement these two interfaces and its methods but the problem is compiler will get confused which method to call as both the methods are having same name.
  • For that, we prefix the method using the interface name and a dot operator, and at the time of calling that the method we are typecasting it so that we can call the correct method.
