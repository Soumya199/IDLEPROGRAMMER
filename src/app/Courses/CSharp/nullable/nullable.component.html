Concept of Boxing,Unboxing and Nullable types in C#

Boxing

Boxing is known as the process of converting a value type to the type object. When the common language runtime (CLR) boxes a value type, it wraps the value inside a System.Object instance and stores it on the managed heap. Boxing is an implicit conversion.

Example

int p=4000;
object obj=p;

Now both the integer variable and the object variable exist on the stack. However, the value of the object resides on the heap.

Unboxing

Unboxing is an explicit conversion from the object type to a value type .

Example

int p=4000;
object obj=p;
int w=(int)obj;

Nullable type in C#

Nullable types represent value-type variables that can be assigned the value of null. Using nullable we can make a variable to hold null value. When ever we are talking about Nullable one question arises why we need a nullable type?. The answer is quite simple, nullable will help us at the time of dealing with database.You cannot create a nullable type based on a reference type because reference types already support the null value.

Example :

int ? program=null;

Here "?" Symbol is used to make program variable of integer type to hold null value. If you write like this
int program=null;
That will give you an error like this "Cannot convert null to 'int' because it is a non-nullable value type".
