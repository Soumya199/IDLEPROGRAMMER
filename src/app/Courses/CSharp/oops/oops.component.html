What is OOP in C#?


OOP stands for Object-Oriented Programming.Object-oriented programming (OOP) is the core ingredient of the .NET framework. OOP is so important that, before embarking on the road to .NET, you must understand its basic principles and terminology to write even a simple program.Understanding OOP concepts can help make decisions about how you should design an application and what language to use.

C# offers full support for OOP including inheritance, encapsulation, abstraction, and polymorphism:

Lets Discuss something about Oops :

A class is like a blueprint of a specific object or we can say class is a collection of objects. In the real world, every object has some color, shape, and functionalities - for example, the Royal Enfield Bike. Royal Enfield is an object of the Bike type. The Bike is a class that indicates some characteristics like speed, color, shape etc. So we can say that 'Bike' is a class, and every single physical Bike of any Company is an object of the Bike class.

In C#, Object is a real world entity In other words, object is an entity that has state and behavior. Object is an instance of a class. All the members of the class can be accessed through object.

Inheritance

Inheritance is the process by which one object acquires the properties of another object.In the language of C#, a class that is inherited is called a base class. The class that performs the inheriting is called a derived class. Therefore, a derived class is a specialized version of a base class. It inherits all of the variables, methods, properties, operators, and indexers defined by the base class and adds its own unique elements

Example : 
                
                    using System;
                    public class Employee
                    {{ '{' }}
                        public float salary = 40000;
                    {{ '}' }}
                    public class Programmer: Employee
                    {{ '{' }}
                        public float bonus = 10000;
                    {{ '}' }}
                    class TestInheritance
                    {{ '{' }}
                        public static void Main(string[] args)
                        {{ '{' }}
                            Programmer p1 = new Programmer();
                            Console.WriteLine("Salary: " + p1.salary);
                            Console.WriteLine("Bonus: " + p1.bonus);
        
                        {{ '}' }}
                    {{ '}' }}
                  Output :
                   
                        Salary: 40000
                        Bonus: 10000

polymorphism

Polymorphism is a Greek word whose meaning is "one name many forms". Polymorphism is one of the core concept of Object Oriented Programming that provides the ability to a class to have multiple implementations with the same name.

There are two types of polymerphism are there let's discus about that

(1) Static polymerphism :-

  • Static polymerphism is also called as compile time polymerphism because the response to a method is determined at the time of compilation
  • C# provides two types of techniques to use static polymerphism They are:

(1)Method overloading

(2)Operator overloading

Method overloading :-

  • Method overloading refers to have more than one method with same name but different parameters
  • in method overloading C# compiler checks the number of parameters passed and the type of the parameter and make the decision of which method to call, if no matching method is found this will throw an error.

Example :-

                
                    using System;
                    public class Employee
                    {{ '{' }}
                        public void Print(int x)
                        {{'{'}}
                            Console.WriteLine(x);
                        {{'}'}}
                        public void Print(string name)
                        {{'{'}}
                            Console.WriteLine(name);
                        {{'}'}}     
                    {{ '}' }}
                    public class Program
                    {{'{'}}
                        public static void Main(string[] args)
                        {{'{'}}
                            Employee empObject=new Employee();
                            empObject.Print(10);
                            empObject.Print("idleprogrammer.com");
                        {{'}'}}
                    {{'}'}}
                  Output :
                   
                        10
                        idleprogrammer.com

(1) Dynamic polymerphism :-

  • Method overriding refers to have more than one method with same name and same parameter.
  • Method overriding is implemented using inheritance, using method overriding it is possible for base class and derive class to have the same method name and the same parameter as discussed in first point.
  • The compiler will decide which method to call at runtime and if no method is found then it throws an error.

Example :-

                
                    using System;
                    public class Calculation
                    {{'{'}}
                        public void Print(int x)
                        {{'{'}}
                            x=x*x;
                            Console.WriteLine( x );
                        {{'}'}}
                    {{'}'}}
                    public class child:Calculation
                    {{'{'}}
                        public void Print(int x)
                        {{'{'}}
                            x=x*10;
                            Console.WriteLine(x);
                        {{'}'}}
                    {{'}'}}
                    public class Program
                    {{'{'}}
                        public static void Main(string[] args)
                        {{'{'}}
                            child empObject=new child();
                            Calculation objectP=new Calculation();
                            objectP.Print(10);
                            empObject.Print(5);
                        {{'}'}}
                    {{'}'}}
                
                  Output :
                   
                        100
                        50
